# Module: Controllers

## Purpose

Controllers are internal (unexported) types that handle specific CRUD operations. Each controller holds a pointer to the parent `Crud` instance and implements one or more HTTP handler methods.

## Controller Overview

| Controller | File | Routes |
|------------|------|--------|
| `entityManagerController` | `entity_manager_controller.go` | `entity-manager`, `home`, (default) |
| `entityCreateController` | `entity_create_controller.go` | `entity-create-modal`, `entity-create-ajax` |
| `entityReadController` | `entity_read_controller.go` | `entity-read` |
| `entityUpdateController` | `entity_update_controller.go` | `entity-update`, `entity-update-ajax` |
| `entityTrashController` | `entity_trash_controller.go` | `entity-trash-ajax` |

## Entity Manager Controller

**File:** `entity_manager_controller.go`

### page(w, r)

Renders the main entity listing page.

**Features:**
- Breadcrumb navigation (Home → Entity Manager)
- "New Entity" button that loads the create modal via HTMX (`hx-get`)
- DataTable with sortable columns from `ColumnNames`
- Per-row action buttons (View, Edit, Trash) conditionally shown:
  - **View:** shown when `FuncFetchReadData != nil`
  - **Edit:** shown when `FuncFetchUpdateData != nil`
  - **Trash:** shown when `FuncTrash != nil`
- Vue.js 3 app (`EntityManager`) mounted on `#entity-manager`
- Server-side pagination (when `PageSize > 0` and `FuncRowsCount` configured)

**Raw HTML support:** Column names and cell values wrapped in `{!! !!}` are rendered as raw HTML.

**Response:** Full HTML page (`Content-Type: text/html`)

**CDN resources added:** jQuery DataTables CSS/JS

## Entity Create Controller

**File:** `entity_create_controller.go`

### modalShow(w, r)

Returns an HTML fragment containing a Bootstrap 5 modal with form fields from `CreateFields` using `github.com/dracory/form`. Designed to be loaded via HTMX.

**Features:**
- Form fields generated by `form.NewForm().Build()`
- "Create & Edit" submit button with JavaScript fetch
- "Close" cancel button
- Modal backdrop

**Response:** HTML fragment

### modalSave(w, r)

Processes entity creation via AJAX.

**Pipeline:**
1. Enforce POST method
2. Check `FuncCreate != nil`
3. Extract field values from request body using `listCreateNames()`
4. Validate required fields (non-empty check)
5. Call `FuncCreate(r, data)`
6. Return JSON with `entity_id` and `redirect_url`

**Response:** JSON (`api.Response`)

## Entity Read Controller

**File:** `entity_read_controller.go`

### page(w, r)

Renders a read-only view of an entity.

**Pipeline:**
1. Extract `entity_id` from query parameters
2. Check `FuncFetchReadData != nil`
3. Call `FuncFetchReadData(r, entityID)` → `[]KeyValue`
4. Render key-value table in a Bootstrap card
5. Append `FuncReadExtras` content (if configured)

**Features:**
- Raw HTML support for keys and values via `{!! !!}`
- Edit and Back navigation buttons
- Error alert if data fetch fails (page still renders with 200)

**Response:** Full HTML page (`Content-Type: text/html`)

## Entity Update Controller

**File:** `entity_update_controller.go`

### page(w, r)

Renders an edit form for an entity.

**Pipeline:**
1. Extract `entity_id` from query parameters
2. Check `FuncFetchUpdateData != nil`
3. Call `FuncFetchUpdateData(r, entityID)` → `map[string]string`
4. Generate form fields from `UpdateFields` with Vue.js `v-model` bindings
5. JSON-encode field values for Vue.js initialization
6. Mount `EntityUpdate` Vue app on `#entity-update`

**Features:**
- "Save" button (saves and redirects to manager or `UpdateRedirectURL`)
- "Apply" button (saves without redirect)
- Image upload via FileReader API
- Trumbowyg WYSIWYG editor for `htmlarea` fields
- Element Plus date picker for `datetime` fields

**CDN resources added:** jQuery DataTables, Trumbowyg CSS/JS, Vue Trumbowyg, Element Plus

**Response:** Full HTML page (`Content-Type: text/html`)

### pageSave(w, r)

Processes entity update via AJAX.

**Pipeline:**
1. Enforce POST method
2. Extract `entity_id` from request body
3. Extract field values using `listUpdateNames()`
4. Validate required fields
5. Call `FuncUpdate(r, entityID, data)`

**Response:** JSON (`api.Response`)

## Entity Trash Controller

**File:** `entity_trash_controller.go`

### pageEntityTrashAjax(w, r)

Processes entity soft-deletion via AJAX.

**Pipeline:**
1. Enforce POST method
2. Extract `entity_id` from request body
3. Check `FuncTrash != nil`
4. Call `FuncTrash(r, entityID)`

**Response:** JSON (`api.Response`)

### pageEntitiesEntityTrashModal()

Generates a Bootstrap modal HTML fragment for the trash confirmation dialog. This modal is embedded in the entity manager page and controlled by Vue.js.

**Returns:** `hb.TagInterface` (not an HTTP handler)
